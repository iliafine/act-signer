# Сервис подписания актов

## Что это
Локальный сервис (FastAPI + браузерный UI) для автоматической вставки
сканов подписей в акты. Работает офлайн: загружаете акт `.docx` или zip с
актами, сервис сам находит блок подписантов, ищет нужные подписи в базе
(по ФИО) и вставляет их «поверх» места для подписи — как будто подписали
от руки. Результат можно получить в **.docx** или **.pdf**.

Отдельно — `registry_batch/batch_sign.py`: батч-режим по реестру (Excel).

## Установка и запуск

### Через GitHub (рекомендуется для переноса на другой компьютер)
Так имена файлов и кодировки переносятся корректно (zip по почте часто
ломает кириллицу в именах).

```
git clone https://github.com/iliafine/act-signer.git
cd act-signer
```

Затем запустить лаунчер (см. ниже). При первом запуске он сам создаст
виртуальное окружение и поставит зависимости.

### Запуск
- **Windows:** двойной клик `start.bat`. Нужен Python 3.10+
  (python.org → при установке отметить «Add Python to PATH»).
- **Mac:** двойной клик `start.command`.

Откроется http://127.0.0.1:8743 в браузере.

**Остановка:** `stop.bat` (Windows) / `stop.command` (Mac).

### Экспорт в PDF (опционально)
Для выгрузки результата в PDF нужен **LibreOffice** (бесплатный,
libreoffice.org). Без него экспорт в `.docx` работает как обычно.
Сервис сам находит LibreOffice в стандартных путях установки —
дополнительной настройки не требуется.

## Структура

```
backend/
  app/main.py        — веб-API (загрузка, подписание, экспорт PDF/DOCX, CRUD подписантов)
  app/docx_signer.py — поиск подписантов в акте + вставка подписи на «полку»
  app/image_utils.py — очистка скана подписи (фон/линии) + усиление (толщина, цвет)
  app/signature_db.py— база подписантов (signatures/db.json)
  requirements.txt   — зависимости Python
signatures/          — подписи (PNG, ASCII-имена sig_<id>.png) + db.json
registry_batch/      — батч-режим по реестру (Excel)
start.bat / start.command — запуск
stop.bat  / stop.command  — остановка
```

База подписантов (`signatures/db.json`) хранит для каждого: ФИО (обязательно),
должность/роль (опционально, для запасного сопоставления), `scale` (размер
росчерка) и `vertical_offset_frac` (сдвиг привязки к полке вниз для
размашистых подписей). Имена файлов подписей — только ASCII (`sig_<id>.png`),
чтобы перенос между ОС не ломался.

## Возможности
- Акты: **.docx** на вход; результат — **.docx** или **.pdf**.
- Один акт или **zip** с множеством актов (на выходе — zip + текстовый отчёт
  по проблемным файлам).
- Сопоставление приоритетно по **ФИО** (надёжнее), должность — как запасной
  вариант.
- Подпись вставляется плавающим изображением «поверх» текста, садится на
  линию-«полку»; размашистые подписи (Коршак и т.п.) настраиваются через
  `scale` и `vertical_offset_frac` в карточке подписанта.
- Очистка сканов: OpenCV (бинаризация/выделение чернил) + усиление штрихов.
